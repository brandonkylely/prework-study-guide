HTML

  • Use HTML to build the structure of your "house".
  • The <head> element contains information about the webpage.
  • Head includes <title> and link to external css style sheet.
  • The <body> element represents the visible content shown to the user.
  • Headers: <h1> through <h6>
  • Sections: <section> through </section>
  • class=" " and id=" " for external css
  • Lists: <ul> or <ol>; <li>

CSS

  • Use CSS to style your "house".
  • Inline CSS: Unique style for a single element within HTML.
  • Internal CSS: Define all <style> elements within HTML.
  • External Style css sheet (.css): Define all <style> elements on a separate file.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Use Git to control changes in a systematic way
  • touch " ": creates new folder
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Git Flow!: git add -A, git commit -m " "... stages changes, commits locally
  • ...git pull origin main, git push origin <insert branch name>... makes sure local branch is up to date, pushes changes to remote repository
  • GitHub-->Create pull request-->Peer Review-->Merge
